Position Property 101

This is an infomational about the values that you can set to the position property and how to use them.

Values Definition Notes
Inherit Takes the position value of its parent element.
  • None
Initial Sets property to its default value--which is static.
  • Makes elements appear in order top to bottom as its written in the code.
  • Z-index is zero.
Relative Element moves relative to its static/default position.
  • Reserves initial space in flow of page.
Absolute Moves relative to its closest positioned(non-static) ancestor/element container.
  • If there is none it moves relative to its initial container.
  • Margins do not collapse with othe margins.
Fixed Moves relative to the broswer window/viewport. (always on the screen)
  • Removed from the content flow.
  • No space is created for this element.
Static Positioned in the order written in the html flow.
  • Default. Can be used as fallback.
  • Position properties (top, right, bottom, lef, z-index) have no affect.
Sticky Positioned in normal flow of the html document in a relative position until scrolled passed, then it stick so the screen until returned to its initial position.
  • Toggles between being relative and fixed.
  • Sticks to nearest element with scrolling behavior. Could inhibit sticky behavior.
  • Internet explorer doesn't support this positioning. And, Safari requires a -webkit-prefix
  • Requires a positioning property to function.
